| **GESTIÓN DEL REPOSITORIO GITHUB Y REGISTRO VERIFICABLE DE LA EVOLUCIÓN A V2.0** | Este criterio evalúa tu capacidad de aplicar control de versiones para almacenar y actualizar el proyecto, evidenciando avances del diseño (CSS, layout y responsive) en el repositorio. | Gestiona el repositorio con commits frecuentes y claros por mejora (CSS base, Flexbox, Grid, responsive), mantiene organización de archivos consistente y README actualizado con evidencia de cambios, demostrando trazabilidad completa de la evolución a v2.0. | Aplica comandos básicos de control de versiones bajo supervisión, utilizando el repositorio en la nube para almacenar y actualizar el código, evidenciando los avances principales de v2.0. | Actualiza el repositorio con commits escasos o poco consistentes, pero suficientes para verificar el avance general del sitio hacia v2.0. | Actualiza de manera intermitente, con dificultades para mantener el repositorio alineado con el estado real del sitio o con evidencia poco clara de cambios. | Actualiza el repositorio de forma incipiente, registrando al menos un commit y un push relacionado con cambios de estilos o maquetación, aunque con evidencia incompleta o poco clara de la evolución del producto hacia v2.0. |
| **VERIFICACIÓN DE RESPONSIVIDAD Y CORRECCIÓN DE FALLAS DE VISUALIZACIÓN DEL SITIO** | Este criterio evalúa tu capacidad para ejecutar pruebas de visualización en distintos tamaños de pantalla (móvil/tablet/desktop) y corrige problemas de diseño y maquetación para asegurar usabilidad. | Verifica sistemáticamente el sitio en múltiples resoluciones con Dev Tools, corrige quiebres de layout y asegura legibilidad, evitando scroll horizontal innecesario y manteniendo consistencia visual en secciones clave. | Ejecuta pruebas guiadas con DevTools responsive y corrige problemas básicos de maquetación para asegurar un funcionamiento visual aceptable en diferentes tamaños de pantalla. | Realiza pruebas limitadas y corrige parcialmente problemas, el sitio es usable, aunque conserva detalles de responsive, alineación o espaciado sin resolver. | Ejecuta pruebas incompletas, persisten fallas visibles que afectan la experiencia del usuario en móvil o tablet, requiriendo guía frecuente. | Realiza una verificación mínima de la visualización del sitio, identificando de manera superficial fallas de maquetación, aunque no logra corregirlas de forma consistente. |
| **ORGANIZACIÓN DE ESTILOS Y ESTRUCTURA DEL SITIO PARA MANTENIBILIDAD Y ESCALABILIDAD** | Este criterio evalúa tu capacidad para diseñar y organizar la estructura del sitio y sus estilos (clases, secciones y layout) para facilitar mantenimiento, consistencia y crecimiento del proyecto. | Organiza el CSS por secciones/componentes con clases consistentes y reutilizables, minimiza duplicación y mantiene una estructura de layout clara (Flexbox/Grid), facilitando la evolución y ajustes futuros del sitio. | Diseña la estructura general del sitio y organiza los componentes principales del layout y estilos según los requerimientos de la versión 2.0, manteniendo coherencia básica del proyecto. | Organiza estilos y layout de forma básica, con inconsistencias en nombres de clases o duplicación moderada, pero permite mantener y ajustar el sitio con esfuerzo razonable. | Organiza estilos de forma poco clara: hay duplicación frecuente y dificultad para ajustar secciones sin generar efectos colaterales, requiriendo guía constante. | Organiza los estilos y layout de forma incipiente, creando una estructura mínima de CSS y aplicándolas parcialmente al sitio, aunque con desorden e inconsistencias, evidenciando un intento inicial de mantener el proyecto. |
