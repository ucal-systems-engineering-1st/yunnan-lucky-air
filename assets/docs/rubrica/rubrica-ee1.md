# **RÚBRICA DE EVIDENCIA EVALUATIVA 1**

| CRITERIO | DEFINICIÓN DEL CRITERIO | SOBRESALIENTE (5 PUNTOS) | NOTABLE (4 PUNTOS) | APROBADO (3 PUNTOS) | EN PROCESO (2 PUNTOS) | INICIO (1 PUNTO) |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **IMPLEMENTACIÓN DEL SITIO WEB (HTML)** | Este criterio evalúa tu capacidad de construir el sitio web en HTML incorporando estructura, navegación, secciones de contenido y formulario, evidenciando funcionamiento completo de la versión 1.0. | Implementa un sitio web completo y funcional con estructura semántica consistente, navegación sin enlaces rotos y contenido íntegro (multimedia, lista y tabla), incorporando formulario operativo con validación HTML5 y coherencia general del producto. | Implementa un sitio web funcional con estructura y navegación operativas, incorporando contenido requerido (multimedia, lista y tabla) y formulario con validación HTML5 básica, siguiendo ejemplos o guías. | Implementa un sitio web parcialmente funcional con estructura y navegación mayormente operativas, incorporando contenido mínimo requerido, presentando errores menores de marcado o consistencia que no impiden el uso general. | Implementa un sitio web incompleto con estructura básica y navegación limitada, con secciones o elementos requeridos incompletos, requiriendo guía frecuente para lograr funcionamiento aceptable. | Implementa un sitio web inicial con estructura mínima en HTML, incorporando al menos una sección y un enlace o elemento básico, aunque presenta errores de marcado y navegación limitada, evidenciando un intento de la versión 1.0 del producto. |
| **GESTIÓN DEL REPOSITORIO GITHUB Y REGISTRO DE AVANCES DEL PROYECTO** | Este criterio evalúa tu capacidad de aplicar control de versiones para almacenar y actualizar el proyecto del sitio web en un repositorio en la nube, evidenciando avances verificables. | Gestiona el repositorio en GitHub con commits frecuentes y claros por avance (estructura, contenido, formulario), mantiene organización de archivos consistente y README completo, evidenciando trazabilidad del desarrollo sin requerir supervisión. | Aplica comandos básicos de control de versiones bajo supervisión, utilizando un repositorio en la nube para almacenar y actualizar su propio código, evidenciando commits y push verificables del avance. | Aplica comandos básicos con apoyo puntual, actualizando el repositorio con commits poco consistentes o escasos, pero suficientes para verificar el avance general del producto. | Aplica comandos de control de versiones de forma intermitente, con dificultades para actualizar el repositorio o mantener evidencia clara del avance del proyecto. | Almacena el proyecto en un repositorio en la nube de forma incipiente, realizando al menos un commit y un push, aunque con errores de procedimiento o falta de claridad en el registro, evidenciando un intento de control de versiones. |
| **VERIFICACIÓN FUNCIONAL DE NAVEGACIÓN, CONTENIDO Y FORMULARIO DEL SITIO** | Este criterio evalúa tu capacidad de realizar pruebas básicas del sitio web para verificar navegación, visualización del contenido y funcionamiento del formulario, corrigiendo errores evidentes. | Verifica sistemáticamente enlaces, secciones y formulario en distintos escenarios de uso, identifica fallas y las corrige oportunamente, dejando el sitio estable y sin errores visibles en el flujo normal. | Ejecuta pruebas estandarizadas siguiendo instrucciones o guías predefinidas para verificar navegación, contenido y formulario, corrigiendo errores comunes para asegurar funcionamiento básico. | Ejecuta pruebas básicas con apoyo puntual y corrige parcialmente errores detectados, el sitio funciona, aunque mantiene detalles menores sin resolver. | Ejecuta pruebas de forma incompleta y requiere guía frecuente para identificar fallos; persisten errores que afectan la navegación o el formulario. | Realiza una verificación mínima del sitio, identificando de manera superficial faltas evidentes, aunque no logra corregirlas de forma consistente. |
| **ESTRUCTURA DEL SITIO Y ORGANIZACIÓN INICIAL DEL PROYECTO** | Este criterio evalúa tu capacidad de diseñar la estructura general del sitio web incorporando secciones y relaciones (navegación-contenido-contacto), organizando los componentes principales del proyecto para su crecimiento. | Diseña una estructura clara y coherente del sitio (jerarquía de secciones, encabezados, ids/clases iniciales) y organiza archivos y recursos de forma consistente, facilitando mantenimiento y evolución del proyecto. | Diseña la estructura general del sitio a partir de los requerimientos del proyecto, organizando los componentes principales (secciones y navegación) de manera comprensible. | Diseña una estructura básica con secciones principales, pero con organización parcialmente consistente o relaciones mejorables entre navegación y contenido. | Diseña una estructura inicial incompleta o poco articulada, requiriendo guía frecuente para organizar secciones y relaciones del sitio. | Diseña una estructura incipiente del sitio incorporando secciones básicas de forma parcial y organiza el proyecto con una estructura mínima de archivos, aunque con desorden e inconsistencias, evidenciando un intento de relacionar navegación y contenido. |
