## **2\. Descripción de la evidencia evaluativa:**

Este trabajo consiste en la construcción de un sitio web versión 1.0 aplicando fundamentos de HTML y control de versiones con Git y GitHub, evidenciando una estructura clara del producto, contenido completo, formulario funcional y registro verificable del avance en el repositorio.

El sitio web deberá estar diseñado para atender específicamente las necesidades de un caso de estudio de alcance internacional que trabajarás a lo largo del ciclo.

Elaborarás y entregarás, de manera grupal (hasta 5 integrantes), un sitio web versión 1.0 incorporando como mínimo:

**a. Estructura HTML completa del sitio**

* Documento HTML con estructura correcta (head, meta viewport, body) y secciones coherentes con el propósito del sitio.

**b. Navegación funcional**

* Menú o enlaces de navegación con accesos a secciones internas (anclas) y enlaces externos verificados (sin enlaces rotos).

**c. Contenido estructurado y multimedia**

* Inclusión de imágenes con atributo alt.
* Una lista (mínimo 5 ítems) y una tabla (mínimo 3x4) integradas en una sección del sitio.

**d. Formulario de contacto**

* Formulario con campos pertinentes (mínimo: nombre, correo, motivo/mensaje).
* Uso de label y validación HTML5 básica (required, type="email").

**e. Evidencia de desarrollo en GitHub**

* Repositorio en GitHub con commits verificables que demuestren el avance del proyecto durante las 4 semanas.
* README con descripción del sitio y guía breve de navegación.

*Recuerda que esta evidencia tiene un valor del 15% de la nota final.*

## **3\. Indicaciones de los entregables:**

**Entregable 1 \- Link del repositorio GitHub (obligatorio)**

El repositorio debe incluir:

* index.html funcional y secciones completas del sitio.
* Carpeta de recursos (por ejemplo: /assets/img o equivalente) si aplica.
* Historial de commits (mínimo 4 commits significativos, recomendación: 1 por semana o por hito).
* README con:
  * propósito del sitio,
  * mapa de secciones,
  * instrucciones básicas para revisar el proyecto (abrir index.html o link de Pages si lo publican).

**Entregable 2 \- Evidencia funcional del sitio (obligatorio)**

El equipo entrega una de estas dos opciones:

* **Opción A (recomendada):** link de GitHub Pages del sitio publicado.
* **Opción B:** en el README incluir instrucciones claras para ejecutar localmente (abrir index.html) y confirmar que el sitio se visualiza correctamente.

*Para EE1 no es requisito usar branches o pull requests; la evidencia principal del control de versiones se valida con commits/push y el repositorio actualizado.*

## **4\. Indicaciones sobre exposición:**

La exposición será grupal (hasta 5 integrantes) y se realizará en la semana 4\.

* **Duración total:** máximo 15 minutos
  * a. 10 minutos: demo del sitio (recorrido \+ prueba de funcionalidades)
  * b. 5 minutos: preguntas del docente y retroalimentación
* **Participación obligatoria:** cada integrante debe intervenir (mínimo 1 minuto) y señalar su aporte (PR/branch/commits o componente trabajado).
* **Apoyo visual:** Canva o PPT (máximo 8 diapositivas) con:
  1. objetivo del sitio,
  2. estructura/arquitectura del proyecto,
  3. flujo GitHub (branches/PR),
  4. calidad (hallazgos y correcciones),
  5. despliegue (Pages),
  6. conclusiones.
