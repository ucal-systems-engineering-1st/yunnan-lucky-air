## **2\. Descripción de la evidencia evaluativa:**

Este trabajo consiste en que demuestres la evolución del proyecto a una versión 2.0 mediante la implementación de CSS para estilización y maquetación, aplicando Flexbox, Grid y criterios de diseño responsive, asegurando una interfaz consistente, usable y verificable en el repositorio.

Elaborarás y entregarás, de manera grupal (hasta 5 integrantes), la versión 2.0 del sitio web, incorporando como mínimo:

**a. CSS externo y estilos base**

* Implementación de hoja de estilos externa (por ejemplo: /css/styles.css) enlazada correctamente.
* Estilos base consistentes: tipografía, tamaños, colores, espaciados y legibilidad (line-height).

**b. Maquetación con Flexbox (layout 1D)**

* Aplicación de Flexbox en componentes clave (por ejemplo: header/navbar y sección de tarjetas/cards).
* Uso correcto de alineación y distribución (justify-content, align-items, gap, flex-wrap si aplica).

**c. Maquetación con CSS Grid (layout 2D)**

* Implementación de Grid en una sección del sitio (por ejemplo: galería o grilla de contenido destacado).
* Uso de grid-template-columns y gap para una grilla ordenada.

**d. Diseño responsive con media queries**

* Implementación de al menos 2 breakpoints (móvil y tablet/desktop) ajustando navegación y secciones maquetadas.
* Verificación de que el sitio sea usable en pantallas pequeñas (sin desbordes relevantes/scroll horizontal innecesario).

**e. Evidencia de evolución en GitHub**

* Repositorio actualizado con commits verificables que demuestren la evolución de v1.0 a v2.0.
* README actualizado con resumen de mejoras de UI/maquetación y cómo probar el sitio.

El sitio web deberá atender específicamente las necesidades del caso de estudio de alcance internacional proporcionado.

*Recuerda que esta evidencia tiene un valor del 25% de la nota final.*

## **3\. Indicaciones de los entregables:**

**Entregable 1 \- Link del repositorio GitHub (obligatorio)**

El repositorio debe incluir:

* CSS externo implementado y aplicado al sitio.
* Evidencia de Flexbox y Grid en secciones identificables.
* Media queries para responsive.
* Historial de commits que evidencie mejoras por hitos (recomendación: mínimo 4 commits significativos durante semanas 5-8).
* README con:
  * "Mejoras v2.0" (estilos base, Flexbox, Grid, responsive).
  * instrucciones para probar visualización en móvil/tablet con DevTools.

**Entregable 2 \- Evidencia funcional del sitio (obligatorio)**

El equipo entrega una de estas dos opciones:

* **Opción A (recomendada):** link del sitio publicado en GitHub Pages.
* **Opción B:** instrucciones en README para ejecución local (abrir index.html) y cómo validar responsive en DevTools.

*Para EE2 todavía no es requisito usar branches o pull requests; se evalúa principalmente la trazabilidad con commits y el repositorio actualizado.*

## **4\. Indicaciones sobre exposición:**

La exposición será grupal (hasta 5 integrantes) y se realizará en la semana 8\.

* **Duración total:** máximo 15 minutos

    2. 5 minutos: preguntas del docente y retroalimentación
